2026-06-16 在 项目开发步骤.md 新增 3.5 节"项目经理模式（Agent 派发与监控）"。

**Why:** 之前规则只定义了任务卡"可直接交给 Agent 团队执行"，但没有规定 Claude 在项目中如何组织 Agent 团队。项目经理模式填补了执行层角色与协作流程的空白——Claude 是项目经理，负责统筹、监控、汇报，具体执行通过 subagent 派发，避免单兵作战、上下文污染、产出冲突。

**How to apply:**
- 进入任何已启用规则的项目后，Claude 默认按 3.5 节行为：先派发、再监控、后汇报
- 触发派发的条件：可拆分、跨领域、耗时、可并行、需隔离上下文
- 派发指令必须包含：任务目标、上下文、交付物、技术约束、边界、验收标准、汇报格式
- 监控点：范围合规、接口一致、进度偏差、质量门禁、文档同步
- 汇报收敛到里程碑，subagent 返回后内部校验不打扰用户
- 同步更新到 模板/CLAUDE.md.template，使新项目初始化自动具备该行为
