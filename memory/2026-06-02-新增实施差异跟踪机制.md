于 2026-06-02 在任务卡管理规范中新增三项变更：

## 变更清单

### 1. implementation-delta.md 模板（新建）
模板路径：`模板/文档模板/implementation-delta.md`
用于记录实际代码实现与任务卡描述的偏差，包含：变更日期、相关模块、变更类型、差异描述、新增字段、涉及文件、影响范围

### 2. README 状态标记体系（完善）
在 `项目开发步骤.md` 2.8.3 配套文档中增加：
- README 状态列，支持三种标记：✅ 代码已交付、📋 任务卡就绪、⚠️ 有增量
- 状态标记图例说明和使用规则
- 将 `implementation-delta.md` 纳入配套文档清单

### 3. 联动规则（新增）
在 `项目开发步骤.md` 2.8.10 中定义任务卡与差异跟踪的联动关系：
- 正常实现 → 状态 ✅，无额外操作
- 有偏差 → implementation-delta.md 追加记录 + README ⚠️ 标记

**Why:** 缺少实现偏差的记录机制，导致开发过程中的变更不可追溯，后续维护困难。

**How to apply:** 每次任务卡实现时，检查是否与任务卡原始描述一致；有偏差时按联动规则操作。
